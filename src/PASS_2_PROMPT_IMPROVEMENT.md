# Pass 2 - Prompt Improvement Pass

## Completed

This pass strengthened the prompt suite created in Pass 1 and verified in Pass 1.5.

The main change is that every generated section now has a clearer commercial reasoning chain rather than soft instruction wording.

## Core Rules Added / Strengthened

### Commercial Impact / Cost / Valuation Sections

Every paragraph must follow:

Event or instruction -> activity blocked or changed -> resource retained, disrupted, remobilised or used differently -> sequence/productivity/attendance effect -> Defined Cost, valuation or loss and expense consequence.

This prevents weak output that jumps straight from an event to money.

### Programme Impact

Every paragraph must follow:

Event -> planned activity/sequence affected -> activity blocked, changed, resequenced or made less productive -> programme consequence or programme risk.

The prompts now avoid unsupported claims of critical delay or invented programme dates.

### Background

Every paragraph must follow:

Event fact -> affected activity/location -> operational consequence -> commercial relevance.

This keeps the section factual but commercially useful.

### Conclusion

The conclusion must close the submission through:

Event/change -> changed method/sequence/resources -> commercial consequence -> entitlement/valuation request -> requested action.

### Commercial Pushback

Each likely objection must follow:

Likely objection -> why the contractor may raise it -> factual/contractual answer -> evidence needed -> negotiation position.

### Rebuttal

The email-ready response must follow:

Acknowledge contractor position -> state why it is not accepted -> identify event/change -> show activity blocked or changed -> show resource/sequence/productivity effect -> answer objection -> state commercial position -> request reassessment.

## Contract-Specific Language

NEC prompts preserve NEC language such as compensation event, Scope, Accepted Programme, Defined Cost and clause assessment.

JCT prompts preserve JCT language such as Variation, Sub-Contract Works, Employer's Requirements, tender basis, valuation, loss and expense and prolongation.

## Output Formatting

All prompts continue to ban markdown, headings, labels, bullets, numbered lists, bold text and phrases wrapped in asterisks.
