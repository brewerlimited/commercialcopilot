# Pass 1.5 — Prompt Wiring Audit

## Purpose

Pass 1 created separate section prompts. Pass 1.5 verifies and hardens the wiring so the generated section outputs are actually captured and exported.

## Confirmed / Patched

### Multi-stage generation

The following agents are called during multi-stage generation and stored into `client_output`:

| Section | Agent | Stored key | Excel mapping |
|---|---|---|---|
| Background | Background Agent | `client_output.background` | Summary, Basis of Change |
| Change to Contract Basis | Change to Contract Basis Agent | `client_output.change_to_contract_basis` | Basis of Change |
| Effect on Defined Cost / Valuation | Effect on Defined Cost / Valuation Agent | `client_output.effect_on_defined_cost` | Basis of Change, Prelims + Fee |
| Programme Impact | Programme Impact Agent | `client_output.effect_on_programme` | Basis of Change, Time Impact |
| Commercial Impact | Commercial Impact Agent | `client_output.commercial_impact` | Summary, Basis of Change, Prelims + Fee |
| Contractual Position | Contractual Position Agent | `client_output.contractual_position` | Summary, Basis of Change |
| Assumptions | Assumptions Agent | `client_output.assumptions` | Basis of Change, Time Impact, Audit |
| Risks & Qualifications | Risks and Qualifications Agent | `client_output.risks_and_qualifications` | Basis of Change, Time Impact, Audit |
| Conclusion | Conclusion Agent | `client_output.conclusion` | Summary, Basis of Change, Time Impact, Audit |

### Commercial Pushback

Commercial Pushback is generated by its own agent and stored under:

- `internal_commercial_intelligence.commercial_pushback`
- `internal_commercial_intelligence.evidence_gaps`
- `internal_commercial_intelligence.strength_summary`
- `internal_commercial_intelligence.internal_risk_notes`

The workbook export now includes all of the above on the `Commercial Pushback` tab when that output is selected.

### Rebuttal

Rebuttal remains a separate post-pack workflow because it requires a contractor response.

It is generated via:

- `app/api/generate-rebuttal/route.ts`
- `getRebuttalPrompt(payload)`

It is not part of initial CE pack generation or the workbook export unless a future export feature is added.

## New diagnostics

Multi-stage generation now attaches:

```ts
multi_stage_section_wiring_audit
```

to the AI draft output.

This records:

- which agent generated each section
- where it is stored
- whether output was generated
- which workbook tab(s) consume it

## Workbook audit tab

The Excel workbook now adds a `Prompt Wiring Audit` sheet when multi-stage diagnostics/audit data is present.

This allows us to confirm in the exported workbook that split prompt outputs were generated and mapped correctly.
